# Concept: parameter sampling (step 1)

Step 1 draws a low-discrepancy sample of ABM parameter combinations and
pairs each combination with a small bank of spatially comparable initial
conditions. The output is a single **sweep manifest** (JSON + Parquet)
that the external step-2 (PhysiCell-running) agent consumes — one row per
simulation it needs to run.

## What gets sampled

Sampling is a two-stage process:

1. **Unit-hypercube draw.** The chosen backend produces an
   `(n_parameter_samples, n_dims)` array of points in `[0, 1]^d`, where
   `d` is the number of `ParameterSpec` entries in the config.
2. **Per-parameter scaling.** Each column `u` is mapped into the
   parameter's `[low, high]` bounds using the spec's `scale`:
   - `scale="linear"`: `value = low + u * (high - low)`.
   - `scale="log10"`: `value = 10 ** (log10(low) + u * (log10(high) - log10(low)))`.

Log scaling is the right choice for parameters that span orders of
magnitude (rate constants, concentrations).

**Worked example.** A spec
`ParameterSpec(name="r_exh", low=1e-4, high=1e-2, scale="log10")`
maps unit values as follows:

| `u`  | scaled value |
| ---- | ------------ |
| 0.0  | 1.0e-4       |
| 0.5  | 1.0e-3       |
| 1.0  | 1.0e-2       |

Each row's distance is uniform in `log10` space, so the sample covers
all three decades evenly rather than crowding into the upper one.

## Sampler backends

Set `SweepConfig.sampler` to one of:

| Backend        | When to pick it                                                                                            |
| -------------- | ---------------------------------------------------------------------------------------------------------- |
| `pyDOE3`       | Default. Classic Latin Hypercube; matches the reference scripts and the LCSS manuscript's methodology.     |
| `scipy-lhs`    | LHS with `scipy.stats.qmc.LatinHypercube` (centred / strength-2 variants available upstream).              |
| `scipy-sobol`  | Low-discrepancy quasi-random sequence. Best for very high `n_parameter_samples` with `d <= 21`.            |
| `scipy-halton` | Alternative quasi-random sequence. Cheap and deterministic; fine for small sweeps and quick sanity checks. |

All four backends are seeded from `SweepConfig.seed`; the same seed and
backend always yield the same draw.

## Initial-condition replicates

Every parameter combination is paired with `n_initial_conditions`
**spatially comparable** starting tissues, generated by
[`tissue_simulator.ReplicateGenerator`][ts]. The generator first packs a
bootstrap tissue, then resamples replicates until each one's pairwise
spatial-interaction statistics fall within `similarity_tolerance`
(~10% by default) of the bootstrap.

Why this matters: the LCSS manuscript constrains initial conditions to
"preserve comparable spatial organization, ensuring that trajectory
differences reflected parameter effects rather than spatial artefacts
introduced by initial conditions." Without this constraint, two
simulations under the same parameter combination could diverge simply
because their starting tissues were structurally different — and the
landscape would conflate parameter sensitivity with spatial noise.

!!! note "Reproducibility patch"
    Upstream `tissue_simulator` instantiates `numpy.random.default_rng()`
    with no argument inside its packer and tissue classes, so it ignores
    the seed passed into `ReplicateGenerator`. `generate_initial_conditions`
    monkey-patches those call sites for the duration of the call with a
    `SeedSequence`-derived wrapper, so the whole sweep is reproducible
    from a single integer seed. See the docstring on
    `tmelandscape.sampling.tissue_init` for the full mechanism.

The generator writes one CSV per replicate
(`ic_0000.csv`, `ic_0001.csv`, ...) with columns
`x, y, z, radius, cell_type, is_boundary`, into the
`initial_conditions_dir` passed to `generate_sweep`.

## The sweep manifest

`SweepManifest.save(stem)` writes two sibling files:

- **`<stem>.json`** — full Pydantic dump: the originating `SweepConfig`,
  the `initial_conditions_dir`, every `SweepRow`, the creation timestamp,
  and the `tmelandscape` version that produced it. This is the canonical
  artefact and the only one `SweepManifest.load` reads.
- **`<stem>.parquet`** — the rows table only, flattened so each
  parameter becomes its own `float64` column. Provided for cheap
  downstream analytics (pandas, DuckDB, Polars). The schema is fixed
  even when `rows` is empty.

The Parquet columns are:

| Column                     | Type    | Meaning                                                                |
| -------------------------- | ------- | ---------------------------------------------------------------------- |
| `simulation_id`            | string  | Unique id, e.g. `sim_000042_ic_007`.                                   |
| `parameter_combination_id` | int64   | Index into the parameter-combination set (`0 .. n_parameter_samples`). |
| `ic_id`                    | int64   | Index of the IC replicate (`0 .. n_initial_conditions`).               |
| `ic_path`                  | string  | Relative path of the IC CSV under `initial_conditions_dir`.            |
| _one per parameter_        | float64 | Scaled parameter value for this row.                                   |

Total row count is `n_parameter_samples * n_initial_conditions` — the
full Cartesian product. This Parquet file is what the external step-2
agent reads to enumerate the simulations it needs to run.

## Code example

```python
from pathlib import Path

from tmelandscape.config.sweep import ParameterSpec, SweepConfig
from tmelandscape.sampling import generate_sweep

config = SweepConfig(
    parameters=[
        ParameterSpec(name="r_exh", low=1e-4, high=1e-2, scale="log10"),
        ParameterSpec(name="r_kill", low=1e-3, high=1e-1, scale="log10"),
        ParameterSpec(name="t_div_hours", low=12.0, high=48.0, scale="linear"),
    ],
    n_parameter_samples=5,
    n_initial_conditions=3,
    sampler="pyDOE3",
    seed=20260513,
)

manifest = generate_sweep(config, initial_conditions_dir=Path("ics"))
manifest.save("sweep_manifest")  # writes sweep_manifest.json + .parquet
```

This yields 15 rows (`5 * 3`), 3 CSVs under `ics/`, and the manifest
pair next to the working directory.

## CLI

The same call from the shell:

```bash
tmelandscape sample sweep_config.json \
    --manifest-out sweep_manifest \
    --ic-dir ics \
    --target-n-cells 500 \
    --similarity-tolerance 0.10
```

`sweep_config.json` is a JSON dump of a `SweepConfig`. The command
prints a JSON summary (manifest paths, row count, IC count) on stdout.

## What's _not_ in step 1

Step 1 stops at the manifest. **Running the PhysiCell simulations
listed in it is the external step 2**, which is intentionally out of
scope for `tmelandscape` (see the [roadmap](../development/ROADMAP.md)).
The manifest is the contract: hand the JSON + Parquet pair plus the IC
directory to the simulation runner, and step 3 picks back up from the
resulting PhysiCell output directories.
